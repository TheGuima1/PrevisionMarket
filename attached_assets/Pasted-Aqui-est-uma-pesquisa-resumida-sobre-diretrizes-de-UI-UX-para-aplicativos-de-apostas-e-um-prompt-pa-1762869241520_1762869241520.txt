Aqui está uma pesquisa resumida sobre diretrizes de UI/UX para aplicativos de apostas e um prompt para ser usado no Replit. Ele orienta as mudanças necessárias para que a página principal mostre, em cada mercado, tanto a probabilidade implícita (SIM/NÃO) quanto o payout (odds), seguindo boas práticas de design.

## Observações de pesquisa

1. **Informação clara e organizada** – Apps de apostas devem mostrar apenas as informações essenciais de forma clara e simplificada. Exibir muitas estatísticas e odds sem organização sobrecarrega o usuário; portanto, é recomendável destacar as opções mais populares e guardar informações secundárias em locais menos proeminentes.
2. **Visual limpo e responsivo** – Uma UI poluída torna a leitura de odds e execução de apostas mais difícil. É importante usar uma paleta de cores consistente, alinhada à marca, e garantir que o layout se adapte bem a diferentes tamanhos de tela.
3. **Uso de cores e tipografia** – Cores e elementos visuais ajudam o usuário a focar no que importa. Uso de cores contrastantes para diferenciar SIM e NÃO, com tipografia consistente, permite identificar rapidamente probabilidades e odds.

## Prompt para o Replit

> **Título:** Mostrar probabilidade e payout/odd em cada cartão de mercado (página principal)
> **Objetivo:** Na página principal (`Home`), cada cartão de mercado deve exibir a probabilidade implícita de SIM e NÃO junto com o payout (odds) de forma clara, sem depender de *tooltips*.
> **Arquivos a editar:** `client/src/components/market-card.tsx` (e, se necessário, componentes relacionados).
> **Instruções detalhadas:**
>
> 1. **Exibir probabilidade e payout**: No componente `MarketCard`, use as funções existentes `getYesPriceFromReserves`, `getNoPriceFromReserves`, `probToOdds` e `formatProbability` para calcular:
>
>    * `yesPrice` e `noPrice` (probabilidades de 0 a 1);
>    * `yesOdds` e `noOdds` (payout decimal via `probToOdds`);
>    * `yesProbabilityLabel` e `noProbabilityLabel` usando `formatProbability`.
> 2. **Alterar o layout das colunas**: Substitua a grade atual de duas colunas que exibe apenas as odds por um layout que mostre, em cada coluna (SIM e NÃO), algo como:
>
>    ```
>    SIM
>    3,0% (25×)
>    ```
>
>    ou seja, coloque a probabilidade (percentual) em destaque e, ao lado ou abaixo, o payout multiplicador (odds). Use tipografia tabular (como já usado para odds) para manter alinhamento.
> 3. **Remover tooltips**: Como a probabilidade agora estará visível, remova as `Tooltip` e `TooltipContent` que apenas mostravam a probabilidade.
> 4. **Estilização**:
>
>    * Utilize cores distintas para SIM e NÃO (por exemplo, `text-primary` ou `text-chart-1` para SIM e `text-destructive` ou `text-chart-2` para NÃO).
>    * Garanta que a paleta siga a identidade visual do app e que o design fique limpo e legível, conforme diretrizes que destacam a importância de uma interface organizada e com esquema de cores consistente.
>    * Mantenha a altura das linhas compacta para não poluir o cartão; use classes Tailwind adequadas (`text-xs` para rótulos, `text-xl` ou `text-2xl` para números).
> 5. **Responsividade**: Certifique-se de que os elementos se ajustem bem em dispositivos menores; se necessário, use classes de responsividade Tailwind (ex. `sm:text-base` etc.).
> 6. **Testar**: Após implementar, rode o projeto (`npm run dev`) e verifique na página principal se cada mercado mostra claramente “SIM 3% (25×)” e “NÃO 97% (1.03×)” (valores exemplificados). O layout deve permanecer limpo e alinhado, sem sobrecarga de informações.

Esse prompt no Replit ajudará a orientar a modificação do componente `MarketCard` para exibir probabilidades e payouts de maneira clara e consistente com as boas práticas de UI/UX citadas.
